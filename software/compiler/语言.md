# 编程语言基础
- 静态和动态的区别
	- 编程语言是否允许编译器决策问题
	    - 允许
			- 编程语言使用静态策略，或者说问题在编译时被决策
				- 比如java、C++
		- 不允许
	        - 问题只允许在执行项目时被决策，称为动态策略 
		        - 或者可以说，在运行时被决策
					- 比如PHP、Python

- 声明的范围
	- 静态范围/词汇范围
		- 只通过项目就能确定声明的范围
	- 动态范围
    - static
	    - 编译器能够确定声明变量在内存中的位置
        - 只有一个类变量

- 环境和状态
	- 在存储里，环境是一个从名字到地址的映射
	- 情况是一个从地址到值的映射

- 静态范围和块结构
	- 声明的范围是由声明的地点而决定的
	- 状态是从地址到值的映射
